Ausführliche Beschreibung

Logger Klasse, um die Logs zu schreiben.

Autor
Nikolai Klatt

Realisiert als Singleton, das heißt nur eine Instanz pro Programminstanz.
Zugriff über getLogger() Methode von überall aus möglich.

Beschreibung der Konstruktoren und Destruktoren

◆ Log()

de.Enigma.Core.Log.Log ( )
private

Privater Konstruktor, zum Erzeugen eines Singelton Objektes.

Der Konstruktor kann nur intern aufgerufen werden, somit kann nur die Methode getLogger() ein Objekt erzeugen. Der Logger als Singelton Objekt zu erzeugen, ist logisch, da nur eine Instanz des Loggers existieren sollte, denn sonst kann es zu Fehlern und Überschneidungen kommen. Der Logger erzeugt auch gleich den FileHandler.

String  error 
)

Methode zum Schreiben von Error-Logs

Parameter
class_Method_Name- Name der Klasse und Methode von der der Aufruf erfolgt ist
error- Error Nachricht, die vom Logger geloggt werden soll

Erstellt über die .dll einen Error-Log der über kritische Fehler innerhalb der Anwendung informieren soll.
Darunter zählen:

  • Exceptions, die durch einen try-catch geworfen werden
  • kritische Fehler, die das System beeinträchtigen können
  • ...
    Der Aufruf erfolgt immer über die static getLogger() Methode: Log.getLogger().e(getClass().getName()+ ".methodName", "warning");

◆ getLogger()

static Log de.Enigma.Core.Log.getLogger ( )
static

Methode zum Erstellen des Singelton Logger-Objektes.

Rückgabe
gibt das logger Singelton Objekt zurück

Erstellt ein Singleton Objekt, sobald die Instanz benötigt wird und gibt, falls das Objekt schon existiert das Objekt zurück. Das stellt sicher, dass es nur einen Logger gibt und dieser von allen Objekten von überall aus genutzt werden kann.

Methode zum Schreiben von Info-Logs

Parameter
class_Method_Name- Name der Klasse und Methode von der der Aufruf erfolgt ist
info- Info Nachricht, die vom Logger geloggt werden soll

Erstellt über die .dll einen Info-Log der über Vorgänge innerhalb der Anwendung informieren soll.
Darunter zählen:

  • generelle Infos über die Programmroutinen, wie zum Beispiel:
    • Routine gestartet
    • Routine erfolgreich beendet
    • Routine startet Subroutine
    • ...
  • Infos zu Programmabläufen
  • ...
    Der Aufruf erfolgt immer über die static getLogger() Methode: Log.getLogger().i(getClass().getName()+ ".methodName", "warning");

Methode zum Schreiben von Warning-Logs

Parameter
class_Method_Name- Name der Klasse und Methode von der der Aufruf erfolgt ist
warning- Warning Nachricht, die vom Logger geloggt werden soll

Erstellt über die .dll einen Warning-Log der über potentiell gefährliche Vorgänge innerhalb der Anwendung informieren soll.
Darunter zählen:

  • potenzielle Exceptions, die nicht durch einen try-catch gecatched werden können
  • potenzielle breakpoints im Code die zu vernachlässigbaren Fehlern führen können
  • potenzielle aber korrigierbare Fehler
  • ...
    Der Aufruf erfolgt immer über die static getLogger() Methode: Log.getLogger().w(getClass().getName()+ ".methodName", "warning");
